Js正则表达

发表于 2016-05-21 | 分类于 Javascript

正则表达式是用于匹配字符串中字符组合的模式。在 JavaScript中，正则表达式也是对象。这些模式被用于 RegExp 的 exec 和 test 方法, 以及 String 的 match、replace、search 和 split 方法。

创建一个正则表达

两种方法：

var reg = /^[a-zA-Z]+[0-9]*\W?_$/gi;
// 使用一个正则表达式字面量，其由包含在斜杠之间的模式组成

var reg = new RegExp(“\w+”)
//使用构造函数RegExp，需要加引号和转义符号。可以使用传入多个参数构造复杂的正则表达

验证方法

  • exec() 方法检索字符串中的指定值。返回值是被找到的值。如果没有发现匹配，则返回 null。
  • test() 方法搜索字符串指定的值，根据结果并返回真或假。

特殊字符

特殊字符 描叙
^ 匹配字符的开头位置；放在方括号中表示取反的意思
$ 匹配字符的结束位置
. 匹配除换行符 \n 之外的任何单字符。
? 匹配一个字符出现零次或者一次 等价于{0,1}
+ 匹配一个字符出现零次或者一次 等价于{1,}
* 匹配一个字符出现零次或者多次 等价于{0,}
w 匹配任意一个字符、数字、下划线字符；等价[A-Za-z0-9_]
W 取小写w相反值；等价[^A-Za-z0-9_]
d 匹配一个数字；等价于[0-9]
D 取反小写d；等价于[^0-9]
s 匹配一个空白字符，包括\n\r\f\t\v等
S 取反小写s
b 匹配一个单词的边界，即字与空白间的位置
B 取反小写b
{n} n是一个非负整数。匹配n次
{n,} n 是一个非负整数。至少匹配n 次
{n,m} m 和 n 均为非负整数，其中n <= m。最少匹配 n 次且最多匹配 m 次

在量词 *、 + 、? 或 {} 的后面如果紧跟?，将会使量词变为非贪婪的（匹配尽量少的字符），而和默认使用的贪婪模式（匹配尽可能多的字符）正好相反

转义字符

转义字符 描叙
\f 换页符
\n 换行符
\r 回车符
\t 制表符
\v 垂直制表符

例子

解释：

  • 第一个括号：匹配在冒号和两个正斜杠前面的任何单词
  • 第二个括号：匹配 / 和 : 之外的一个或多个字符
  • 第三个括号：匹配冒号后面的零个或多个数字
  • 第四个括号：匹配不包括 # 或空格字符的任何字符序列
